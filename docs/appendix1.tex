\section{实验环境}

论文中的离线演化实验和基准评测均运行在一套四节点 Kubernetes 集群上，其中包含 1 个控制平面节点和 3 个工作节点。集群使用 Kubeadm 搭建，版本为 v1.33.9。所有节点均运行 Ubuntu 24.04.4 LTS，容器运行时为 Docker 29.3.0，并通过 cri-dockerd 接入 Kubernetes。节点之间使用内网地址通信，不配置外部暴露地址，便于在受控环境下持续开展故障注入与重复实验。

从网络侧看，集群使用 Flannel v0.27.4 作为 CNI 插件，对应 DaemonSet 部署在 kube-flannel 命名空间中，节点后端模式为 VXLAN，各节点分别分配独立的 Pod 网段。集群部署了 OpenEBS 本地卷组件作为 CSI 插件，提供 openebs-hostpath 与 openebs-device 两个存储类。实验中需要持久化的微服务应用均使用 OpenEBS Local PV 进行存储，且未额外注册独立的 CSI Driver 对象。为了支持故障注入，集群中还部署了 Chaos Mesh v2.8.1，以 DaemonSet 形式运行在 chaos-testing 命名空间中，提供针对 Pod、容器和网络层面的声明式故障注入能力。最后，为了支持指标采集与监控，集群中部署了 Prometheus Node Exporter、Prometheus Operator 等组件。
